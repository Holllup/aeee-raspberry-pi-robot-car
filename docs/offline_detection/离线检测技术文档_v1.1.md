# 离线检测技术文档（黑线偏离 + 紫色方块）

## 1. 目标

该工具用于离线分析视频中两项任务的检测效果：

- 黑线巡线偏离值（`line_error_px`）
- 右侧紫色方块检测命中（`purple_found`）

适用场景：

- 不上车先看算法效果
- 回放视频检查误检/漏检
- 快速对比参数调整前后表现

---

## 2. 文件说明

目录中的核心文件：

- `line_purple_offline_detector.py`
- `离线检测技术文档.md`（本文档）

脚本运行后会自动输出：

- `*_analysis.csv`：逐帧检测数据
- `*_annotated.mp4`：带标注回放视频
- `*_summary.json`：汇总指标

---

## 3. 检测流程

每一帧按以下顺序处理：

1. 黑线检测
2. 紫色方块检测
3. 写入逐帧 CSV
4. 绘制可视化标注并写入输出视频

### 3.1 黑线检测（偏离值）

- 在图像下方主 ROI 先检测黑线
- 若主 ROI 丢线，切换恢复 ROI 检测
- 取最大轮廓质心作为线中心 `line_center_x`
- 偏离值计算：
  - `line_error_px = line_center_x - frame_center_x`

结果字段：

- `line_found`：当前帧是否检测到线
- `line_mode`：`primary` 或 `recovery`
- `line_error_px`：偏离像素值（右偏为正，左偏为负）

### 3.2 紫色方块检测

- 仅在画面右侧 ROI 检测（减少误检）
- HSV 阈值筛选紫色区域
- 形态学开闭运算去噪
- 轮廓筛选（面积、近似四边形、填充率、solidity）
- 输出最大候选框

结果字段：

- `purple_found`：当前帧是否检测到紫块
- `purple_x/y/w/h`：紫块边界框
- `purple_area`：候选区域面积

## 5. 输出指标解读

`summary.json` 关键指标：

- `line_detect_rate`
  - 黑线检测命中率（越高越好）
- `purple_detect_rate`
  - 紫块检测命中率（目标出现时应高）
- `line_error_abs_mean`
  - 偏离绝对值均值（越小越稳）
- `line_error_abs_p95`
  - 偏离绝对值 P95（反映大偏差尾部）

建议结合 `annotated.mp4` 一起看：

- 命中率高但偏离大：跟踪有线但控制可能偏激
- 紫块命中率低：先看目标是否真实出现在右侧 ROI

## 6. 调参建议

如果黑线漏检：

- 调整 `BLACK_THRESHOLD`
- 适当增大 `MIN_LINE_AREA`
- 调整主/恢复 ROI 的中心和高度比例

如果紫块漏检：

- 调整 `PURPLE_LOWER_HSV / PURPLE_UPPER_HSV`
- 适当放宽 `PURPLE_MAX_ASPECT_ERROR`
- 适当降低 `PURPLE_MIN_FILL_RATIO`

如果紫块误检：

- 提高 `PURPLE_MIN_AREA`
- 提高 `PURPLE_MIN_SOLIDITY`
- 缩小 `PURPLE_RIGHT_REGION_RATIO`（更聚焦右侧）

---

## 7. 建议上线策略

在车上实时触发停车时，不建议单帧触发。建议：

- 连续 `2~3` 帧 `purple_found=1` 才触发
- 触发后增加冷却时间（例如 3 秒）
- 停车 5 秒后自动恢复巡线

这样能显著减少抖动和偶发误检导致的误触发。
