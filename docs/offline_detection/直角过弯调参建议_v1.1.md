# 直角过弯调参建议 v1.1

## 目标

在“巡黑线 + 右侧紫块检测”的基础上，让小车更稳地通过直角弯和大角度弯。

## 核心策略

不要在直角弯继续按普通误差跟踪，改用状态机：

1. `track`：正常按误差差速控制。
2. `corner_pivot`：当下方 ROI 丢线且历史误差持续偏大，短时强制转向。
3. `search_turn`：仍未回线时，降低速度继续按记忆方向搜索。
4. 回线后立即切回 `track`。

## 触发条件建议

- 黑线丢失：`line_found == 0`
- 最近误差大：`abs(last_nonzero_error) >= CORNER_ERROR_THRESHOLD`
- 丢线时间短：`time_since_line <= CORNER_PIVOT_TIMEOUT_SECONDS`

以上条件同时满足，进入 `corner_pivot`。

## 参数推荐（先从这一组开始）

- `CORNER_ERROR_THRESHOLD = 34`
- `CORNER_PIVOT_SPEED = 28`
- `CORNER_PIVOT_TIMEOUT_SECONDS = 0.72`
- `RECOVERY_TURN_SPEED = 26`
- `RECOVERY_FORWARD_SPEED = 4`
- `RECOVERY_TURN_TIMEOUT_SECONDS = 1.15`
- `TURN_DIRECTION_MEMORY_SECONDS = 0.80`
- `RECOVERY_ROI_CENTER_RATIO = 0.74`
- `RECOVERY_ROI_HEIGHT_RATIO = 0.40`

## 调参顺序

1. 先固定基础直线参数（`BASE_SPEED`、`STEERING_GAIN`）。
2. 只调直角触发阈值和 `corner_pivot` 速度/超时。
3. 再调 `search_turn` 速度，避免冲出赛道。
4. 最后调恢复 ROI，提升回线成功率。

## 常见现象与修正

- 到直角后停住不转：
  - 降低 `CORNER_ERROR_THRESHOLD`
  - 提高 `CORNER_PIVOT_SPEED`
  - 适当增大 `CORNER_PIVOT_TIMEOUT_SECONDS`

- 到直角后左右抖动：
  - 增大 `TURN_DIRECTION_MEMORY_SECONDS`
  - 提高 `TURN_DIRECTION_LOCK_THRESHOLD`

- 能转过去但经常冲线：
  - 降低 `CORNER_PIVOT_SPEED`
  - 降低 `RECOVERY_TURN_SPEED`
  - 缩短 `RECOVERY_TURN_TIMEOUT_SECONDS`

- 回线慢：
  - 将 `RECOVERY_ROI_CENTER_RATIO` 再上移一点（比如 `0.76`）
  - 增大 `RECOVERY_ROI_HEIGHT_RATIO`

## 与离线检测联动建议

用离线脚本先观察直角附近的指标：

- `line_found` 连续丢失帧数
- `line_error_px` 在入弯前的峰值
- 紫块是否与转弯时序冲突

如果入弯前 `|line_error_px|` 峰值很高但仍不触发转弯，优先降低 `CORNER_ERROR_THRESHOLD`。

## 实车上线建议

- 紫块停车触发采用连续帧确认（`2~3` 帧）+ 冷却时间（`>=3s`）。
- 先在中速验证（`BASE_SPEED` 24~28），稳定后再提速。
- 每改一组参数只跑一圈，记录现象再继续。
