# 树莓派小车车辆控制技术文档

## 1. 目标

本文档总结当前这台小车在树莓派上的控制方式，包括：

- UART 串口控制方式
- 轮子通道映射
- 常用控制命令
- Python 测试代码
- 使用 SSH 直接执行的方法

本文档基于实际联机测试结果整理，不仅参考了 `车辆控制基础代码.md`，也结合了树莓派实机单轮测试结果。

## 2. 通信方式

当前小车使用树莓派通过 UART 与底层控制板通信。

- 串口设备：`/dev/ttyS0`
- 波特率：`57600`
- Python 串口库：`pyserial`
- 典型初始化：

```python
import serial

ser = serial.Serial('/dev/ttyS0', 57600, timeout=0.1, write_timeout=0.1)
```

## 3. 已验证命令格式

根据基础代码和实测，当前常用命令如下。

### 3.1 停车

```text
#ha
```

Python 写法：

```python
stop = b'#ha'
```

### 3.2 同时设置方向和速度

命令格式：

```text
#ba + 4个方向字符 + 4个电机的速度字节
```

例如：

```text
#baffff
```

表示四个通道都按 `f` 方向运动，然后后面再跟四组 16 位速度值。

### 3.3 方向字符

已验证有效的方向字符：

- `f`：forward
- `r`：reverse

说明：

- 不建议使用 `s` 作为“停止”方向位。
- 要让某个轮子停住，建议保留合法方向字符，直接把该轮子的速度写成 `0`。

## 4. 四个轮子的真实通道映射

经过单轮测试，已经确认当前四个速度槽位的真实顺序如下：

1. 右前
2. 右后
3. 左前
4. 左后

也就是说，命令：

```python
b'#baffff' + bytes([v1,0, v2,0, v3,0, v4,0])
```

其中：

- `v1` 控制右前轮
- `v2` 控制右后轮
- `v3` 控制左前轮
- `v4` 控制左后轮

## 5. 单轮测试结论

以下代码和观察结果已经确认：

```python
# 右前

# 右后
test2 = b'#baffff' + bytes([0,0, 40,0, 0,0, 0,0])
```

## 6. 常用动作命令

下面的动作建立在当前通道映射之上。

### 6.1 前进

四个轮子都前进：

```python
forward = b'#baffff' + bytes([40,0, 40,0, 40,0, 40,0])
```

### 6.2 后退

四个轮子都后退：

```python
backward = b'#barrrr' + bytes([40,0, 40,0, 40,0, 40,0])
```

### 6.3 左转原地

右侧轮前进，左侧轮后退：

- 右前、右后：`f`
- 左前、左后：`r`

因此命令为：

```python
left_spin = b'#baffrr' + bytes([36,0, 36,0, 36,0, 36,0])
```

### 6.4 右转原地

右侧轮后退，左侧轮前进：

- 右前、右后：`r`
- 左前、左后：`f`

因此命令为：

```python
right_spin = b'#barrff' + bytes([36,0, 36,0, 36,0, 36,0])
```

### 6.5 柔和左转

右侧快，左侧慢：

```python
soft_left = b'#baffff' + bytes([40,0, 40,0, 20,0, 20,0])
```

### 6.6 柔和右转

左侧快，右侧慢：

```python
soft_right = b'#baffff' + bytes([20,0, 20,0, 40,0, 40,0])
```

## 7. 最小前进测试代码

下面这段代码会让小车前进约 2 秒，然后停车：

```python
import serial
import time

ser = serial.Serial('/dev/ttyS0', 57600, timeout=0.1, write_timeout=0.1)

forward = b'#baffff' + bytes([40,0, 40,0, 40,0, 40,0])
stop = b'#ha'

for _ in range(6):
    ser.write(stop)
    ser.flush()
    time.sleep(0.08)

for _ in range(12):
    ser.write(forward)
    ser.flush()
    time.sleep(0.08)

time.sleep(2.0)

for _ in range(6):
    ser.write(stop)
    ser.flush()
    time.sleep(0.08)

ser.close()
print('FORWARD_DONE')
```

## 9. 调试建议

如果命令发出后小车没有反应，优先检查以下几项：

- 串口是否正确：当前实测使用的是 `/dev/ttyS0`
- 波特率是否为 `57600`
- TX/RX 是否交叉连接
- 树莓派与控制板是否共地
- 是否先发送过 `#ha` 清理旧状态
- 是否把“停止”误写成了非法方向字符

## 10. 当前结论

当前这台车的控制已经可以按 UART 正常工作，最关键的结论是：

- 通信端口是 `/dev/ttyS0`
- 波特率是 `57600`
- 四轮顺序是 `右前 -> 右后 -> 左前 -> 左后`
- 通过 `#ha` 和 `#ba.... + 速度字节` 可以完成基础动作控制
