# 巡线 V1 技术文档

## 1. 概述

本版本对应脚本：

- `D:\Onedrive\OneDrive - The University of Nottingham Ningbo China\桌面\树莓派\line_following_minimal.py`

目标能力：

- 黑线巡线（主 ROI + 恢复 ROI）
- 提前转角（多层 ROI 曲线入口预测）
- 右侧紫色方块触发停车（停 3 秒后自动继续）
- 电机高频独立控制线程（`CONTROL_HZ = 120`）

## 2. 系统架构

当前实现是双线程架构：

1. 视觉线程（主循环）
2. 电机控制线程（固定频率）

### 2.1 视觉线程职责

- 相机采图并缩放：`1280x720 -> 640x360`
- 检测黑线中心点与轮廓
- 采样多层 ROI，推断弯道入口方向
- 检测右侧紫色方块
- 将检测结果写入共享状态 `ControlState`

### 2.2 电机线程职责

- 以固定周期运行：`interval = 1.0 / CONTROL_HZ`
- 从 `ControlState` 读取最新状态
- 根据状态机计算输出模式并发串口命令
- 与视觉帧率解耦，保证误差变化后更快生效

## 3. 硬件与通信

### 3.1 串口通信

- 串口候选：`/dev/serial0`, `/dev/ttyAMA0`, `/dev/ttyAMA10`, `/dev/ttyS0`
- 波特率：`57600`
- 控制协议：
  - `#ha`：急停
  - `#ba + 4方向位 + 4路16位速度`

### 3.2 电机分组

代码中使用：

- 右侧轮组：`RIGHT_MOTOR_INDEXES = (0, 1)`
- 左侧轮组：`LEFT_MOTOR_INDEXES = (2, 3)`

`set_tank_drive(left_speed, right_speed)` 会把左右速度写入四个电机槽位并发送。

## 4. 视觉检测逻辑

### 4.1 黑线检测

流程：

1. ROI 裁剪
2. 灰度 + 高斯滤波
3. 二值化（黑线反相）
4. 形态学开闭操作
5. 找最大有效轮廓并计算质心

关键参数：

- `BLACK_THRESHOLD = 80`
- `MIN_LINE_AREA = 180`
- 主 ROI：
  - `PRIMARY_ROI_CENTER_RATIO = 0.58`
  - `PRIMARY_ROI_HEIGHT_RATIO = 0.24`
- 恢复 ROI：
  - `RECOVERY_ROI_CENTER_RATIO = 0.74`
  - `RECOVERY_ROI_HEIGHT_RATIO = 0.40`

### 4.2 提前转角（入口预测）

额外采样多层 ROI：

- `CURVE_SAMPLE_ROI_CENTER_RATIOS = (0.52, 0.62, 0.72, 0.82)`
- `CURVE_SAMPLE_ROI_HEIGHT_RATIO = 0.12`

根据采样点计算：

- `spread`（横向展开）
- `slope`（上下层中心漂移）
- `curvature`（曲率近似）

输出：

- `direction`: `-1/0/+1`
- `kind`: `lost/straight/gentle/sharp`
- `score`: 弯道置信度

入口预测会以短时记忆写入控制状态，在丢线转向阶段作为辅助方向。

## 5. 紫色方块停车逻辑

检测范围：

- 画面右侧区域：`PURPLE_RIGHT_REGION_RATIO = 0.42`

颜色与几何过滤：

- HSV 阈值：`PURPLE_LOWER_HSV` 到 `PURPLE_UPPER_HSV`
- 最小面积、实心度、填充率、长宽比误差

触发条件：

- 连续命中帧数 `PURPLE_CONFIRMATION_HITS = 2`
- 触发冷却 `PURPLE_TRIGGER_COOLDOWN_SECONDS = 2.0`

动作：

- 进入暂停状态 `pause_active`
- 停车 `PURPLE_ACTION_HOLD_SECONDS = 3.0`
- 时间到后自动恢复巡线

## 6. 控制状态机（电机线程）

电机线程每周期读取状态后按优先级执行：

1. `pause_active`：停车（紫色触发）
2. 有新线且在线有效窗口内：按 `error` 跟踪
3. 丢线短保持：`RECOVERY_HOLD_SECONDS`
4. 角点/入口辅助原地转：`CORNER_PIVOT_TIMEOUT_SECONDS`
5. 搜索转向：`RECOVERY_TURN_TIMEOUT_SECONDS`
6. 超时停车

方向来源优先级：

1. 入口预测记忆方向（有效期内）
2. 最近恢复方向锁定
3. 历史误差符号

## 9. 现象与调参建议

### 9.1 响应慢

优先调整：

- 提高 `CONTROL_HZ`（如 120 -> 160）
- 降低 `PROCESS_SIZE` 以减轻视觉负担
- 适度提高 `STEERING_GAIN`

### 9.2 过弯冲出

优先调整：

- 增大 `ENTRY_MEMORY_SECONDS`
- 降低 `ENTRY_ASSIST_SCORE_THRESHOLD`
- 增大 `CORNER_PIVOT_TIMEOUT_SECONDS`

### 9.3 抖动或蛇形

优先调整：

- 增大 `DEADBAND_PIXELS`
- 降低 `STEERING_GAIN`
- 检查 `BLACK_THRESHOLD` 和光照稳定性

### 9.4 紫色误触发

优先调整：

- 提高 `PURPLE_MIN_AREA`
- 提高 `PURPLE_CONFIRMATION_HITS`
- 收紧 HSV 范围

## 10. 验证建议（切换到 Pi 5B 后）

建议按以下顺序验证：

1. 空载跑 `--headless`，观察稳定性
2. 跑带预览模式，确认 `ENTRY` 与 `Purple` 叠加信息
3. 对比 `CONTROL_HZ=120/160` 的转向手感
4. 做 3 次固定赛道重复测试，记录冲线成功率
