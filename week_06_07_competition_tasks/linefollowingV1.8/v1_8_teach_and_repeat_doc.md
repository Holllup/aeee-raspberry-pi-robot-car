# 巡线小车 V1.8 录像与回放 (Teach & Repeat) 模式使用指南

## 一、核心原理
V1.8 移除了以前硬编码的避障写死时间（转弯2秒、直行4秒）。它现在会尝试从本地加载 `detour_script.json`。
- **如果有这个文件**：小车会在遇到避障标志时，像“看录像”一样，自动重放脚本里记录的所有速度指令和延时。
- **如果没有这个文件**：小车会进入**录制模式（Recording Mode）**停止等待，此时启动了本地的 UDP 服务（端口 `9999`），允许你使用电脑前台遥控接管驾驶。

## 二、使用步骤

### 1. 准备工作
确保树莓派上没有旧的干扰脚本：
如果你想录制新的路径，请先在树莓派执行 `rm detour_script.json`。

### 2. 启动巡线主程序
和平常一样，在后台启动 V1.8 小车（可以用你的 `.bat` 或 `nohup` 脚本）：
```bash
python3 line_following_v1_8_teach_and_repeat.py --arm --headless &
```

### 3. 开始录制 (Teach)
当小车正常巡线遇到**紫色避障标志**时，如果没有剧本，小车会自动停下来。此时，在你的 SSH 终端运行以下命令：
```bash
python3 remote_control.py
```
这会劫持你在终端的按键。
- 长按 **W、A、S、D** 来前后左右驾驶小车，绕出障碍物并回到赛道。
- 松开按键小车会继续跑吗？不会，遥控代码中设置的是发出单独指令，按键时运动。
- 当小车回到赛道黑线上并且你满意了目前的姿态，请按 **回车键 (Enter)**。
- 此时控制台会提示 `Saved detour script`，同时小车会自动恢复正常巡线状态。

### 4. 自动回放 (Repeat)
下次小车再跑到避障标志前，一旦识别成功，无需任何人工干预！
它会自动读取刚生成的 `detour_script.json`，丝毫不差地将你的速度和停留时长重演一遍，最后平滑地切回寻找黑线的模式。

## 三、调试建议
- 录制时尽量**平稳打方向**，不需要极快，求准不求快。
- 如果录制失败或者路线不满意，直接 `Ctrl+C` 退出遥控，然后删掉 `detour_script.json` 重新录一次即可。
