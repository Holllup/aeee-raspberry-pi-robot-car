# 巡线 v1.1 技术文档（自动日志版）

## 1. 本版本目标

本版本用于定位“检测到紫色但未停车”的问题，核心改动是：

- 检测到紫色后立即急停（连续发送 `#ha`）
- 每次运行自动保存视频和日志
- 默认每次运行 20 秒，结束后自动停

主程序文件：

- `line_following_minimal.py`

## 2. 关键硬件配置

### 2.1 UART（车控）

- 波特率：`57600`
- 端口候选优先级：
  - `/dev/ttyAMA0`
  - `/dev/serial0`
  - `/dev/ttyAMA10`
  - `/dev/ttyS0`

### 2.2 电机顺序（实测）

速度字节顺序为：

1. 右前
2. 右后
3. 左前
4. 左后

### 2.3 LCD1602（BCM）

- RS: `GPIO22`
- E: `GPIO17`
- D4: `GPIO25`
- D5: `GPIO18`
- D6: `GPIO24`
- D7: `GPIO23`

### 2.4 舵机

- 引脚：`GPIO27`
- 紫色命中时角度：`90`
- 恢复角度：`180`

## 3. 紫色触发停车逻辑

当右侧区域识别到紫/红方块后：

1. 立即触发急停（连续发送 `#ha`）
2. 进入暂停窗口（默认 5 秒）
3. LCD 显示 `Red Dot`
4. 舵机转 90 度，暂停结束后恢复 180 度

## 4. 自动日志与视频（新增）

每次运行都会自动创建时间戳目录，默认路径：

- `/home/jacob/line_follow_runs/run_YYYYMMDD_HHMMSS/`

目录内包含：

- `run_video.h264`：本次原始录像
- `events.csv`：逐帧事件日志
- `summary.json`：本次运行汇总

`events.csv` 关键字段：

- `t_s`：相对运行时间
- `line_found`：是否检测到线
- `purple_found`：是否检测到紫/红块
- `purple_trigger`：该帧是否触发停车
- `purple_pause_active`：是否处于停车暂停状态

## 5. 运行命令

默认 20 秒自动停（推荐）：

```bash
python3 /home/jacob/line_following_minimal.py --arm
```

无窗口运行：

```bash
python3 /home/jacob/line_following_minimal.py --arm --headless
```

指定运行时长（例如 30 秒）：

```bash
python3 /home/jacob/line_following_minimal.py --arm --duration-seconds 30
```

指定日志根目录：

```bash
python3 /home/jacob/line_following_minimal.py --arm --session-root /home/jacob/Desktop/runs
```

## 6. 排障建议

如果“看见紫色仍不停”，优先检查：

- `events.csv` 中是否出现 `purple_found=1`
- 是否出现 `purple_trigger=1`
- 是否进入 `purple_pause_active=1`

判断方法：

- `purple_found=0`：视觉阈值问题（颜色/光照/ROI）
- `purple_found=1` 但 `purple_trigger=0`：触发条件或冷却条件问题
- `purple_trigger=1` 但车不停：UART/主板执行链路问题
